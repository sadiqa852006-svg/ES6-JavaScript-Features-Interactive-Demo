Feature 01

Arrow Functions

Concise syntax for function expressions with lexical this binding.

⚡
Syntax Comparison
Traditional vs arrow function syntax
// Traditional function add(a, b) { return a + b; } // Arrow — implicit return const add = (a, b) => a + b; // Single param — no parens const square = n => n * n; // No params const greet = () => "Hello!";
🎯
Lexical this
Arrow functions inherit this from the enclosing scope
class Timer { constructor() { this.seconds = 0; } start() { // 'this' stays correct ✓ setInterval(=> { this.seconds++; }, 1000); } }

Creating Promises
const fetchData = new Promise((resolve, reject) => { setTimeout(() => { const ok = Math.random() > 0.3; ok ? resolve({ data: "✓ Success!" }) : reject(new Error("✗ Failed")); }, 1000); }); fetchData .then(res => console.log(res.data)) .catch(err => console.error(err.message)) .finally(() => console.log("Done"));

Chaining & Promise.all
// Sequential chain getUser(1) .then(user => getPosts(user.id)) .then(posts => render(posts)) .catch(err => console.error(err)); // Parallel execution Promise.all([ fetchUsers(), fetchProducts(), ]).then(([users, products]) => { // Both resolved ✓ });

async function loadUser(id) { try { const user = await fetchUser(id); const posts = await fetchPosts(user.id); return { user, posts }; } catch (err) { console.error(`Error: ${err.message}`); } } // Parallel with await const [a, b] = await Promise.all([ taskA(), taskB() ]);

Feature 04

Destructuring

Extract values from arrays and objects into variables with elegant syntax.

📦
Object Destructuring
const user = { name: "Keanu", age: 39, address: { city: "NYC" } }; // Extract & rename const { name: fullName, age } = user; // Nested + default const { address: { city }, role = "user" } = user; // Rest pattern const { name, ...rest } = user;
🗂️
Array Destructuring
const rgb = ["red", "green", "blue"]; // Basic const [first, second] = rgb; // Skip elements const [,, third] = rgb; // Swap variables! let a = 1, b = 2; [a, b] = [b, a]; // Default + rest const [x = 10, ...others] = [5, 20, 30];
🔀
Function Params
// Destructure in params function greet({ name, age = 18 }) { return `Hi ${name}, age ${age}`; } // Array param const first = ([a]) => a; // Mixed nesting const coords = ({ pos: [x, y] }) => `x:${x} y:${y}`;

Feature 05

Modules

Split code into reusable, encapsulated files with import/export syntax.

📁 math.js
export const add = (a,b) => a+b; export const PI = 3.14159; export default class MathLib {};
→
📁 app.js
import MathLib, { add, PI } from './math.js'; add(2, 3); // 5 PI; // 3.14159
📤
Export Types
// Named exports export const PI = 3.14159; export function square(x) { return x*x; } export class Vector { ... } // Default (one per file) export default function main() { ... } // Re-export export { foo } from './foo.js';
📥
Import Patterns
// Named import { PI, square } from './math.js'; // Alias import { square as sq } from './math.js'; // Namespace import * as MathLib from './math.js'; // Dynamic const mod = await import('./module.js');

Feature 06

Template Literals

Multi-line strings with embedded expressions using backtick syntax.

📝
Basic & Multi-line
const name = "World"; // Interpolation const msg = `Hello, ${name}!`; // Expression inside ${ } const tax = `Tax: ${(price * 1.2).toFixed(2)}`; // Multi-line — no \n needed const html = ` <div> <h1>${title}</h1> </div> `;
🏷️
Tagged Templates
// Custom tag function function highlight(strings, ...vals) { return strings.reduce((acc, str, i) => acc + str + (vals[i] ? `<mark>${vals[i]}</mark>` : "" ), "" ); } highlight`Hello ${name}, age ${age}.`;
